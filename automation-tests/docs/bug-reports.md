# Sample Bug Reports

## BUG-001: Checkout Form Allows User to Continue Without Postal Code

**Status:** Sample bug report  
**Severity:** Medium  
**Priority:** High  
**Environment:** Windows 11, Chromium  

### Preconditions

User is logged in and has one product in the cart.

### Steps to Reproduce

1. Open the cart.
2. Click Checkout.
3. Fill First Name.
4. Fill Last Name.
5. Leave Postal Code empty.
6. Click Continue.

### Actual Result

User is able to continue to the next checkout step.

### Expected Result

Application should display validation error for missing postal code.

### Notes

This is a sample bug report created for learning.

## BUG-002: Error Message Is Not User Friendly

**Status:** Sample bug report  
**Severity:** Low  
**Priority:** Medium  
**Environment:** Windows 11, Chromium  

### Preconditions

User is on the login page.

### Steps to Reproduce

1. Enter valid username.
2. Enter invalid password.
3. Click Login.

### Actual Result

Generic error message is displayed.

### Expected Result

Error message should clearly explain that login credentials are invalid.

### Notes

This is a sample bug report created for learning.
